Default Trigger Scenarios

This demo uses the standard trigger defaults, shortens idle to 45 seconds, and opts into prefetch points so auction timing is visible in the event log. Use the controls below or interact with the browser as described.

  • Bottom-of-page Click Bottom-of-page Zone, or scroll until depth reaches 90%. The auction prefetches at 70%.
  • Return-to-top Click Arm Return-to-top, then Return to Top. The auction prefetches while scrolling upward through 50%, and the trigger fires when you return to the top 10%.
  • Idle time Leave the page open. The auction prefetches after 30 seconds, then the trigger fires after 45 seconds.
  • Tab focus return Switch to another browser tab for at least 1 second, then come back. This tests hidden-to-visible visibility changes.
  • App focus return Keep this tab visible, switch to another app/window for at least 1 second, then return. This tests blur-to-focus without conflicting with tab return.
  • Custom setup and manual Custom Setup Trigger dispatches the configured publisher event. Manual Trigger calls pbjs.exitAds.trigger().

Suppressed triggers, no-bid attempts, frequency caps, and the hard 30s display gate are shown in the Event Log.
